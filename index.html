Overview

How the system detects & reacts

The KY-026 flame sensor module pairs with an Arduino Uno to deliver real-time, multi-modal fire alerts. The moment a flame enters the detection zone, three simultaneous alert channels activate instantly.

🔴 Detection Mechanism

The KY-026 contains a photodiode tuned to the 760–1100 nm infrared spectrum emitted by open flames. Its integrated LM393 comparator evaluates the signal and outputs a digital LOW (active-low logic) when a flame is within its 60° viewing angle.

⚡ Arduino Response

The firmware polls the sensor's digital pin (D7) every 200 ms. On detecting LOW, it simultaneously drives the buzzer (D8) HIGH for audible alert, LED (D13) HIGH for visual indicator, and updates the I2C LCD with "FIRE DETECTED!" and intensity level.

📊 Analog Intensity

The KY-026's analog output (A0) provides a 0–1023 ADC value representing IR radiation intensity. This maps to a 0–100% fire proximity percentage displayed on the LCD's second line for real-time awareness of flame distance.

🛡️ Fail-Safe Design

Active-low logic ensures fail-safe behavior — a disconnected or faulted sensor triggers alert state rather than silently failing. All channels (buzzer, LED, LCD) deactivate instantly when the flame is removed, with 200 ms debounce to prevent flicker.

Components

Parts & Materials

All components are available in standard Arduino starter kits or electronics lab workspaces.

🤖

Arduino Uno R3

ATmega328P — 16 MHz

Main microcontroller. 14 digital I/O, 6 analog inputs. Reads the sensor and drives all alert outputs.

🔥

KY-026 Flame Sensor

IR Photodiode + LM393

Detects open flame at 760–1100 nm. 60° detection angle. Dual digital + analog output. Adjustable via onboard potentiometer.

🔊

Passive Buzzer

5V, ~2.7 kHz

Produces audible alert tone when fire is detected. Driven directly from D8. No transistor required at 5V logic.

💡

LED Indicator (Red)

5mm, 2V, 20mA

Visual fire alert. Connected to D13 via a 220Ω current-limiting resistor to protect the GPIO pin.

🖥️

16×2 I2C LCD

PCF8574 — I2C addr 0x27

Displays real-time status and fire intensity via I2C (only SDA + SCL wires). Saves 6 GPIO pins vs parallel wiring.

⚡

Resistor 220Ω

1/4W, ±5% Tolerance

Series current-limiting resistor for the LED. Prevents excessive current through D13 and protects the LED from burnout.

🔌

Jumper Wires

M-M and M-F Sets

M-M wires for breadboard rows, M-F for connecting KY-026 and LCD module pins to the breadboard rails.

📋

Breadboard

830-point Half-size

Solderless prototyping base. Power rails distribute 5V and GND cleanly to all components.

🔋

USB / DC Power

USB-B or 9V DC Barrel

USB from PC for programming and testing. Optional 9V DC barrel connector for standalone deployment away from a computer.

📄 sketch.ino Arduino C++
#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// ─── Pin Definitions ──────────────────────────────────────────────
#define FLAME_DIGITAL  7   // KY-026 DO pin  → Arduino D7 (Button in Wokwi)
#define FLAME_ANALOG   A0  // KY-026 AO pin  → Arduino A0 (Potentiometer in Wokwi)
#define BUZZER         8   // Passive buzzer → Arduino D8
#define LED_PIN        13  // LED (220Ω)     → Arduino D13

// ─── LCD: I2C address 0x27, 16 cols, 2 rows ──────────────────────
LiquidCrystal_I2C lcd(0x27, 16, 2);

// ─── Setup ───────────────────────────────────────────────────────
void setup() {
  pinMode(FLAME_DIGITAL, INPUT_PULLUP);
  pinMode(BUZZER,        OUTPUT);
  pinMode(LED_PIN,       OUTPUT);

  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0); lcd.print("Fire Alert Sys  ");
  lcd.setCursor(0, 1); lcd.print("  Initialising..");
  delay(2000);
  lcd.clear();
}

// ─── Main Loop ────────────────────────────────────────────────────
void loop() {
  // KY-026 active-LOW: LOW = flame detected
  // In Wokwi: press the button to simulate flame detection
  int flameDigital = digitalRead(FLAME_DIGITAL);
  int flameAnalog  = analogRead(FLAME_ANALOG);

  // Map analog: lower ADC = stronger/closer flame = higher %
  int intensity = map(flameAnalog, 1023, 0, 0, 100);
  intensity = constrain(intensity, 0, 100);

  if (flameDigital == LOW) {
    // ── Flame Detected ────────────────────────────────────────────
    lcd.setCursor(0, 0); lcd.print("FIRE DETECTED!  ");
    lcd.setCursor(0, 1);
    lcd.print("Intensity: ");
    lcd.print(intensity);
    lcd.print("%   ");

    // Buzzer beep pattern
    digitalWrite(BUZZER, HIGH);
    delay(100);
    digitalWrite(BUZZER, LOW);
    delay(50);
    digitalWrite(BUZZER, HIGH);
    delay(100);
    digitalWrite(BUZZER, LOW);
    delay(50);

    digitalWrite(LED_PIN, HIGH);

  } else {
    // ── No Flame ─────────────────────────────────────────────────
    lcd.setCursor(0, 0); lcd.print("No Fire Detected");
    lcd.setCursor(0, 1); lcd.print("System: SAFE    ");

    digitalWrite(BUZZER,  LOW);
    digitalWrite(LED_PIN, LOW);
  }

  delay(200);
}

#include <Wire.h>
#include <LiquidCrystal_I2C.h>

// ─── Pin Definitions ─────────────────────────────────────────
#define FLAME_DIGITAL  7   // KY-026 DO pin  → Arduino D7
#define FLAME_ANALOG   A0  // KY-026 AO pin  → Arduino A0
#define BUZZER         8   // Passive buzzer → Arduino D8
#define LED_PIN        13  // LED (220Ω)     → Arduino D13

// ─── LCD: I2C address 0x27, 16 cols, 2 rows ──────────────────
LiquidCrystal_I2C lcd(0x27, 16, 2);

// ─── Setup ───────────────────────────────────────────────────
void setup() {
  pinMode(FLAME_DIGITAL, INPUT);
  pinMode(BUZZER,        OUTPUT);
  pinMode(LED_PIN,       OUTPUT);

  lcd.init();
  lcd.backlight();
  lcd.setCursor(0, 0); lcd.print("Fire Alert Sys  ");
  lcd.setCursor(0, 1); lcd.print("  Initialising..");
  delay(2000);
  lcd.clear();
}

// ─── Main Loop ───────────────────────────────────────────────
void loop() {
  // KY-026 is active-LOW: LOW = flame detected
  int flameDigital = digitalRead(FLAME_DIGITAL);
  int flameAnalog  = analogRead(FLAME_ANALOG);

  // Map analog: lower ADC = stronger flame = higher %
  int intensity = map(flameAnalog, 1023, 0, 0, 100);

  if (flameDigital == LOW) {
    // ── Flame detected ──────────────────────────────────────
    lcd.setCursor(0, 0); lcd.print("FIRE DETECTED!  ");
    lcd.setCursor(0, 1);
    lcd.print("Intensity: ");
    lcd.print(intensity);
    lcd.print("%   ");
    digitalWrite(BUZZER,  HIGH);  // Buzzer ON
    digitalWrite(LED_PIN, HIGH);  // LED ON
  } else {
    // ── No flame ────────────────────────────────────────────
    lcd.setCursor(0, 0); lcd.print("No Fire Detected");
    lcd.setCursor(0, 1); lcd.print("System: SAFE    ");
    digitalWrite(BUZZER,  LOW);   // Buzzer OFF
    digitalWrite(LED_PIN, LOW);   // LED OFF
  }

  delay(200);  // 200 ms poll — prevents LCD flicker
}
How It Works

Step-by-Step System Flow

A complete walkthrough of the detection pipeline from IR emission to multi-modal alert output.

1

IR Emission

The KY-026 module's built-in IR LED continuously emits modulated infrared light across its 60° sensing cone. Open flames emit strong IR radiation in the 760–1100 nm spectrum, making them highly distinguishable from ambient room light.

2

Reflection & Detection

The integrated IR photodiode receiver picks up reflected and radiated infrared energy from a flame source. The stronger the flame or the closer it is, the lower the analog output voltage (higher ADC value on AO pin).

3

LM393 Comparator Decision

The onboard LM393 dual comparator evaluates the photodiode signal against a reference threshold set by the potentiometer. When the IR signal exceeds the threshold, the comparator drives the DO (digital output) pin LOW — active-low logic.

4

Arduino Polling

The Arduino's loop() function calls digitalRead(D7) every 200 ms. A LOW reading triggers the alert sequence. Simultaneously, analogRead(A0) captures the 0–1023 intensity value, which is mapped to 0–100% for LCD display.

5

Multi-Modal Alert Activation

On flame detection, three channels fire simultaneously: digitalWrite(D8, HIGH) activates the passive buzzer for audible alarm; digitalWrite(D13, HIGH) lights the red LED; and the I2C LCD updates with "FIRE DETECTED!" and real-time intensity percentage.

6

Instant Deactivation

When the flame is removed, the sensor's DO pin returns HIGH. On the next 200 ms poll, all three alert channels are driven LOW simultaneously, the LCD displays "System: SAFE", and the system returns to standby — ready for the next detection event.

Timeline

Project Development Journey

A chronological walkthrough of the full development process — from concept to working prototype.

Phase 1 — Project Conception

Identifying the Problem

Identified the need for an affordable, real-time fire detection solution using widely available Arduino components. Researched fire detection methods — IR sensing was chosen for its fast response time, digital simplicity, and low cost compared to gas or thermistor-based sensors.

Research
Phase 2 — Component Selection

Choosing the Right Hardware

Selected the KY-026 flame sensor for its dual digital/analog output, integrated LM393 comparator, and 760–1100 nm spectral sensitivity perfectly matched to open flame IR emissions. Chose Arduino Uno for straightforward I2C and GPIO support.

Hardware Design
Phase 3 — Circuit Design

Schematic & Pin Mapping

Designed the wiring layout: KY-026 DO to D7 for digital detection, AO to A0 for intensity, buzzer to D8, LED to D13 via 220Ω resistor, and 16×2 I2C LCD to A4/A5. I2C bus chosen to preserve digital GPIO pins for potential future expansion.

Circuit Design
Phase 4 — Firmware Development

Writing the Arduino Sketch

Developed the firmware with a 200 ms polling loop reading both digital and analog sensor outputs. Implemented concurrent alert activation (buzzer + LED + LCD), active-low detection logic, and analog-to-percentage mapping for fire intensity display on the LCD's second row.

Software Development
Phase 5 — Virtual Simulation

Testing on Wokwi Simulator

Loaded the circuit and firmware into the Wokwi online simulator. Validated the active-low logic by toggling the digital flame sensor input. Confirmed LCD output updates, buzzer activation, and LED state transitions all behaved correctly before touching real hardware.

Simulation
Phase 6 — Hardware Assembly

Breadboard Prototype Build

Assembled all components on a breadboard following the wiring schematic. Used power rails for clean 5V and GND distribution. Uploaded the sketch via Arduino IDE 2.x using USB. Performed initial power-on test — LCD splash screen confirmed correct I2C address and communication.

Assembly
Phase 7 — Calibration

Sensitivity Tuning

Adjusted the KY-026's onboard potentiometer to set the optimal detection threshold. Tested with a lighter flame at distances of 10 cm, 20 cm, and 30 cm. Fine-tuned the threshold to eliminate false positives from sunlight while maintaining reliable detection at up to 25 cm.

Calibration
Phase 8 — Final Testing & Documentation

System Validation & Write-up

Ran comprehensive tests: repeated flame detection cycles, stress-tested the 200 ms polling loop, verified simultaneous alert activation, and confirmed clean deactivation on flame removal. Documented all findings, circuit diagrams, and code comments for this project showcase.
